# FieldFlow

A mobile-first field service management app (CRM) built for a landscaping company. Replaces three separate tools and a Signal group chat with one unified platform that enforces job accountability through mandatory photo documentation, inventory logging, and a hard completion gate that blocks job submission until all requirements are met.

**Live Demo → [Field Flow](https://contractor-pro-five.vercel.app/)**

---

## The Problem It Solves

Field techs were not accurately logging the materials used on jobs, costing the company significant money in untracked inventory. FieldFlow enforces accountability by requiring techs to:

- Upload before and after photos (minimum 2 each)
- Log every inventory item used on the job
- Complete a job-type-specific checklist
- Add job notes

None of these can be skipped. The app blocks submission until all four pass.

## Features

### Admin
- **Dashboard** — live stats: active jobs, pending reviews, completed today, low stock alerts
- **Jobs** — create jobs with type (tree / irrigation / sod / other), assign to a tech, set a color, manage bid status and amount; filter by status, type, or tech
- **Job Detail** — view all photos (before / during / after), inventory used, checklist answers, notes; approve or request changes with a note
- **Inventory** — manage main stock, set low-stock thresholds, assign items to techs
- **Techs** — manage tech profiles and color assignments
- **Analytics** — jobs by status (pie chart), jobs by type (bar chart), top inventory usage, tech performance table

### Tech (mobile-first)
- **Dashboard** — personal job queue sorted by urgency; inventory summary
- **Job Detail** — advance status (assigned → on the way → in progress), upload photos from camera or gallery, log inventory used, fill out job checklist, write notes
- **Submit for Review** — triggers client-side + server-side validation; shows exactly what's missing if blocked
- **Change Requests** — orange banner shows the manager's note when changes are requested; re-submit after fixing
- **Inventory** — view own stock levels with usage history per item

### System
- Role-based routing enforced at the middleware level (server-side)
- Real-time updates via Supabase channels — job changes appear live without refresh
- Images compressed client-side before upload (max 1MB / 1920px)
- Email notifications on job submission, approval, and change requests (Resend)
- Job number auto-generated by database trigger (`JOB-0001`, `JOB-0002`, …)
- All photos stored in Supabase Storage with signed URLs (not public)
- Full audit trail via `job_status_history` table

## Database Schema

Key tables:

| Table | Purpose |
|---|---|
| `profiles` | Extends `auth.users` — stores role, display name, color |
| `inventory_items` | Master item list with main stock and low-stock threshold |
| `tech_inventory` | Per-tech stock allocations |
| `jobs` | Job records — status, type, color, bid info, locked state |
| `job_photos` | Before / during / after photos linked to jobs |
| `job_inventory` | Materials logged per job |
| `job_checklists` | Type-specific checklist answers per job |
| `job_status_history` | Full audit trail of every status change |
| `inventory_transactions` | Audit trail of inventory movements |

Row Level Security is enabled on every table. Techs can only see their own jobs and inventory. Admins see everything.
